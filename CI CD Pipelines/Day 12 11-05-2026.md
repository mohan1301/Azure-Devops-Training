Self learning
To gather all the requirements and build an application... we use dotnet as backend language.
In case of Linux, we use maven/gradle
Config file is called .csproj and for java it is called pom.xml
Dependency mgmt of all the libraries is taken care by nuget/dotnet and maven central for Java
Build command is dotnet build and mvn clean install for Java
Output .dll/.exe and .jar/.war

Build is different from run...
run command is dotnet run and java -jar

maven is used to build the base source code of the Java application like creating classes, config files etc. maven is a build and dependency management tool

pom.xml file contains project information(group id, artifactid, version)
dependencies will be managed in dependencies block

To create a basic Java Application from terminal
mvn archetype:generate

mvn clean --> remove old build, create new build
mvn compile --> Compiles the code
mvn test --> run test cases
mvn package --> create jar/war
mvn install --> install to local
