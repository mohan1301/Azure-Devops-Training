Links means Shortcuts
Types of Links -> Sim links, Hard links

Sim link means having a linked file from Original file. If any changes made to linked file, original file will get reflected. 
If original file is deleted, no use of having the linked file as original file itself is deleted.
ln -s source_file path.txt linked_file path.txt => ln means link, s means Sim Link
To remove sim link -> rm si linked_file path.txt

Hard Link -> When we create a link from Original file to Linkedfile, even though you delete the original file, linkedfile file works as it should. As they both are independent.
Reason: when we create a file in Linux, it creates a inode value for storing all the data. So, even though you change in one file, it is reflecting in inode. inode is nothing but Metadata.
inode means Index node. inode contains metadata.
metadata includes file type, permission, owner, group id, file size, time stamps, no of links

To make Hard link, we dont need to give -s as we give for Sim link. Directly, give the ln hardlink.txt hardlinkshortcutfile.txt
To know the inode values of the files, ls -li => long listing inode values

We can use the same Sim link and Hard link concepts for folders as well.

cp command to copy the files content into another file
For directories, we need to use -r which is recursive mode

mv command to move the files to another location
