There are 4 merging strategies
1. Merge No Fast Forward(Non Linear PR Strategy) - Preserving Full commit history(Non Linear). Merge bubble will be shown - Enterprise level
2. Squash(Linear PR Strategy) - Combines all the commits into one commit - Used in Messy commits situation
3. Rebase+FastForward - Linear and No merge commit, No branch traces and it looks like continuous commits - Used in Single developer/Simple changes
4. Semi Linear Merge(Rebase+ No FastForward) - Non Linear - Hybrid + PR traces
